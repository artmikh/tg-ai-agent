# Первый запуск и АВТОРИЗАЦИЯ

Pyrogram должен авторизоваться под твоим номером телефона. В Docker это сделать чуть сложнее, поэтому мы сделаем это локально, чтобы файл сессии упал в папку data, а потом запустим контейнер.

Установи зависимости локально (у тебя на компе должен быть Python):
bash
---
cd tg-ai-agent
pip install -r requirements.txt
---

Запусти скрипт локально один раз:
bash
---
python agent.py
---

В консоли появится запрос от Телеграм:
Введи свой номер телефона (тот, с которого будешь читать каналы).
Введи код подтверждения из Телеграм.
Если у тебя включена двухфакторная аутентификация — введи пароль.

Проверь папку data:
После успешного входа скрипт начнет работу. Нажми Ctrl+C, чтобы остановить его. В папке data на твоем компьютере должен был появиться файл user.session и возможно agent_db.sqlite.
Теперь Телеграм запомнил это устройство.

# Сборка и запуск в Docker
Теперь, когда у нас есть user.session в папке data, мы можем запаковывать всё в Docker.

Выполни в папке проекта:

bash
---
docker-compose up -d --build
--build — соберет образ по Dockerfile.
-d — запустит в фоновом режиме (detached).
---

# Проверка работы:

bash

--- Посмотреть логи агента
---
docker-compose logs -f
---

Ты должен увидеть в логах:
Юзербот запущен!
Планировщик запущен. Агент работает.
Парсинг канала: channel1...

Если ИИ найдет совпадение, он отформатирует его и отправит в твой канал ботом!

Развертывание на внешнем сервере
Чтобы перенести это на удаленный сервер (VPS):

Скопируй всю папку tg-ai-agent на сервер (через SCP, SFTP или Git).
Критически важно: Скопируй файл data/user.session! Иначе на сервере Телеграм снова попросит авторизацию, а в Docker в фоновом режиме ты код не введешь.
На сервере выполни: docker-compose up -d --build.
