🤖 Telegram AI Monitor Agent
Универсальный ИИ-агент для автоматического мониторинга Telegram-каналов. Агент парсит указанные каналы, с помощью нейросети семантически анализирует текст (понимая синонимы, опечатки и сленг), форматирует подходящие сообщения по вашему шаблону и публикует их в ваш Telegram-канал.

Решение полностью упаковано в Docker, легко настраивается через конфиг-файлы и готово к распространению и развертыванию на любом сервере.

✨ Ключевые возможности
Семантический поиск: Поиск не по точному совпадению слов, а по смыслу. Найдет "свч", "микроволновку" и "микроволновую печь", если вы ищете "микроволновка".
ИИ-форматирование: Нейросеть извлекает суть (товар, цена, контакты) и оформляет сообщение по вашему шаблону.
Гибкая настройка: Все параметры (промпты, расписание, модели, шаблоны) вынесены в конфиг-файлы. Не нужно лезть в код.
Безопасная архитектура: Чтение чужих каналов идет от вашего аккаунта (MTProto), а публикация — от имени Бота (Bot API). Ваш аккаунт защищен.
Защита от банов: Настраиваемые паузы между чтением каналов и лимиты сообщений для безопасной работы.
Docker-Ready: Легко запускается на любом VPS, данные (история и сессия) сохраняются при перезапусках через Docker Volumes.
⚙️ Как это работает (Архитектура)
Юзербот (Pyrogram): Подключается к Telegram под вашим номером телефона (как обычный клиент) и читает новые сообщения из списка каналов.
Фильтр (OpenAI): Отправляет текст сообщения в LLM с промптом-фильтром. ИИ решает, подходит ли сообщение под ваши ключевые слова.
Форматировщик (OpenAI): Если сообщение подходит, ИИ извлекает данные и возвращает их в формате JSON.
Шаблонизатор: Скрипт подставляет JSON-данные от ИИ в ваш Markdown-шаблон из config.yaml.
Бот (Pyrogram): Авторизуется по токену бота и отправляет готовое сообщение в ваш канал.
База данных (SQLite): Запоминает ID обработанных сообщений, чтобы не было дублей.
🛠 Предварительные требования

Перед началом работы вам необходимо получить следующие данные:

API_ID и API_HASH (для чтения каналов):
Зайдите на my.telegram.org и авторизуйтесь.
Перейдите в раздел API development tools и создайте приложение (заполните любые данные).
Скопируйте App api_id и App api_hash.

BOT_TOKEN (для публикации):
Напишите в Telegram боту @BotFather.
Создайте нового бота (/newbot) и скопируйте полученный токен.
Добавьте бота в ваш канал в качестве Администратора с правом публикации сообщений.

CHANNEL_ID (ID вашего канала):
Перешлите любое сообщение из вашего канала боту @getmyid_bot.
Скопируйте ID (он начинается с -100, например -1001234567890).

OPENAI_API_KEY:
Зарегистрируйтесь на platform.openai.com и создайте API-ключ.


🚀 Быстрый старт
Шаг 1. Подготовка файлов
Склонируйте репозиторий или скачайте архив и распакуйте на вашем сервере.

Убедитесь, что на сервере установлены Docker и Docker Compose.

Шаг 2. Настройка переменных (.env)
Откройте файл .env и вставьте ваши данные, полученные на этапе подготовки:

# Данные аккаунта для ЧТЕНИЯ каналов
API_ID=ваш_api_id
API_HASH=ваш_api_hash

# Данные бота для ПУБЛИКАЦИИ в ваш канал
BOT_TOKEN=ваш_токен_бота
MY_CHANNEL_ID=-100ваш_id_канала

# Ключ OpenAI
OPENAI_API_KEY=sk-ваш_ключ_openai

# Настройки мониторинга
TARGET_CHANNELS=channel_name1,channel_name2,@username_channel
KEYWORDS=микроволновка,свч,печь

Шаг 3. Настройка логики и шаблонов (config.yaml)
Откройте файл config.yaml, чтобы настроить расписание, промпты и шаблон вывода. Вы можете использовать синтаксис Markdown в шаблоне.

yaml

parsing:
  schedule_minutes: 60       # Как часто запускать (в минутах)
  messages_per_channel: 50   # Сколько сообщений читать за раз
  delay_between_channels: 5  # Пауза в секундах между каналами

ai:
  model: "gpt-4o-mini"      # Используемая модель (gpt-4o-mini дешевая и быстрая)

# Шаблон сообщения (поддерживает Markdown)
output_template: |
  🔥 **Найдено предложение!**
  📦 **Товар/Услуга:** {product}
  💰 **Цена:** {price}
  📍 **Локация/Контакт:** {contact}
  📝 **Детали:** {details}

# Промпты (можно адаптировать под свои нужды)
prompts:
  filter: |
    Ты — анализатор текста. Определи, относится ли текст сообщения к следующим темам или ключевым словам: {keywords}.
    Учитывай синонимы, опечатки, сленг и аббревиатуры.
    Ответь ТОЛЬКО одним словом: YES (если относится) или NO (если не относится).
    Текст: {text}

  formatter: |
    Ты — редактор. Извлеки из текста суть и оформи её строго по заданным полям.
    Если каких-то данных нет, пиши "Не указано".
    Верни ответ СТРОГО в формате JSON с ключами: "product", "price", "contact", "details".
    Текст для обработки: {text}
Шаг 4. Первая авторизация (ВАЖНО!)
При первом запуске необходимо создать файл сессии для вашего аккаунта. Это делается один раз. Скрипт запустится в интерактивном режиме, запросит код из Telegram и сохранит сессию в папку data/.

Выполните в терминале (находясь в папке проекта):

bash

docker-compose run --rm tg-agent python auth.py
Скрипт попросит номер телефона, привязанный к аккаунту. Введите его.
Затем введите код подтверждения из Telegram.
Если у вас включена двухфакторная аутентификация, введите пароль.
После сообщения "Авторизация успешна!" нажмите Ctrl+C или дождитесь закрытия.
Шаг 5. Запуск агента
Теперь, когда сессия создана, запускаем агента в фоновом режиме:

bash

docker-compose up -d
Готово! Агент начал работу. Первая проверка каналов произойдет сразу, а затем будет повторяться каждые schedule_minutes.

📋 Управление агентом
Просмотр логов в реальном времени:
bash

docker-compose logs -f
Перезапуск агента (например, после изменения .env или config.yaml):
bash

docker-compose restart
Примечание: изменения в config.yaml подхватятся при перезапуске. Если вы меняли .env, нужно пересоздать контейнер: docker-compose up -d.
Остановка агента:
bash

docker-compose down
⚠️ Важные замечания и безопасность
Риск бана Telegram: Парсинг каналов несет риски временной блокировки аккаунта. Не ставьте слишком маленькие паузы (delay_between_channels < 3 сек) и не читайте сотни каналов каждые 5 минут. Умеренность — ключ к успеху.
Хранение данных: Файлы базы данных (agent_db.sqlite) и сессии (user.session) хранятся в папке data на вашем сервере (проброшены через Docker Volume). При обновлении Docker-образа они не удалятся.
Передача проекта: Образ Docker не содержит ваших личных данных. Вы можете безопасно передавать собранный образ другим людям. Им потребуется лишь пройти Шаг 2 и Шаг 4 со своими данными.
Стоимость OpenAI: Агент использует модель gpt-4o-mini. Она очень дешевая, но при больших объемах сообщений следите за балансом в личном кабинете OpenAI.
